Worked examples

Common analyses
step by step.

Once you have config.sh sourced (see the overview for first-run setup), each of these runs is a one-line command. The wrappers do the joining, logging, and resume bookkeeping for you.

Every wrapper writes a run.log with the exact command, tool versions, and timing, plus a .done sentinel that lets you re-run with new flags without redoing finished steps. Pass --force to override.

Example 01 · characterize

What is this protein, and where does it occur in the Mollusca?

Given a single (or few) protein sequence(s), run all three search modes — BLAST, DIAMOND, and hmmsearch against the mollusc-revised HMMs — in parallel, and roll the best hits into a single report keyed by query. 

wrappers/mg_characterize.sh -q query.fa -o characterize_out -t 8

Outputs:

  • characterize_out/blast/hits_with_species.tsv — BLAST hits joined to species metadata
  • characterize_out/diamond/hits_with_species.tsv — DIAMOND hits
  • characterize_out/hmm/hits_with_species.tsv — per-sequence HMM matches
  • characterize_out/summary.tsv — per-query best hit per method, side-by-side
  • characterize_out/report.html — self-contained HTML rollup, openable in any browser
Tip. For large query sets, drop one of the search modes with --skip diamond (BLAST already covers it) or --skip blast (DIAMOND is faster). The HMM scan is the unique value-add — keep it.
Example 02 · phylogeny

Build a phylum-wide tree of a gene family from reference seeds.

Given a small reference FASTA (one or several seed sequences for the family you care about), iteratively recruit homologs across all of MolluscaGenes, align with MAFFT, trim with ClipKit, infer a tree with IQ-TREE 2, prune rogues with TreeShrink, re-align, infer the final tree, and rename tips from species codes to binomials. 

wrappers/mg_place.sh \
    -q nachr_seeds.fa \
    -o nachr_tree \
    --search diamond \
    --iterations 4 \
    --mafft-mode einsi \
    --clipkit-mode kpic-smart-gap \
    --bb-final 10000 \
    -t 16

Key outputs:

  • nachr_tree/iter1..iter4/hits.tsv — every iterative search round, fully traceable
  • nachr_tree/alignment.fa — MAFFT alignment of the recruited set + seeds
  • nachr_tree/aligned.fa.clipkit — ClipKit-trimmed alignment
  • nachr_tree/iqtree.iqtree — IQ-TREE log; round-1 model selection
  • nachr_tree/treeshrink/run/output.fasta — TreeShrink-pruned set
  • nachr_tree/final.contree — final consensus tree (with bootstraps)
  • nachr_tree/final.names.contree — same tree with species-code prefixes replaced by binomials
Performance. A 4-round DIAMOND-based run with 16 threads on a typical workstation takes ~2 hours wall time end-to-end on a small reference set, dominated by the MAFFT and IQ-TREE rounds. For BLAST-based searches budget ~6 hours. --skip-treeshrink halves the runtime if you don't need rogue-tip removal.
Resume-safe. Each step writes a .step_<name>.done sentinel — you can re-run the command after changing later-stage flags (model, ClipKit mode) and only the affected steps will recompute.
Example 03 · extract

Pull every protein from a single class for a downstream analysis.

Run an analysis on, say, every cephalopod protein in the database (28 species, ~2.8 M sequences in v0.1) without writing any glue code. 

wrappers/mg_extract.sh \
    -s Cephalopoda --by class \
    -o cephalopoda_proteins \
    -d aa --merge \
    -t 8

The selector accepts:

  • A single code: -s Octbim
  • A comma-separated list: -s Octbim,Sepoff,Eupber
  • A binomial: -s "Octopus bimaculoides" --by binomial
  • A taxonomic rank: -s Cephalopoda --by class (also works with order, family, phylum)
  • A file: -s @my_codes.txt (one token per line)

Outputs:

  • cephalopoda_proteins/extracted_aa.fa — single merged FASTA (because of --merge)
  • cephalopoda_proteins/extracted_species.tsv — per-species count of what was actually extracted
First-run cost. The first mg_extract against a fresh database builds an accession-by-species cache (~2 minutes, ~600 MB under metadata/_cache/). Subsequent runs are I/O-bound and complete in seconds.
Example 04 · HMM scan

Scan a proteome with the mollusc-revised HMMs.

Given any protein FASTA — your own assembly, a single species pulled with mg_extract, or anything else — run hmmsearch with the full TIAMMAt-revised HMM bundle and join hits to species metadata. Here we use the two together: 

wrappers/mg_extract.sh -s "Octopus bimaculoides" --by binomial -o obim --merge
wrappers/mg_hmmsearch.sh -q obim/extracted_aa.fa -o obim_hmm -t 8

For Pfam gathering thresholds (the conservative default many curators prefer), use --cut-ga instead of -E: 

wrappers/mg_hmmsearch.sh -q obim/extracted_aa.fa -o obim_hmm --cut-ga -t 8

Or scan with a single HMM rather than the full bundle: 

wrappers/mg_hmmsearch.sh \
    -q obim/extracted_aa.fa \
    -o obim_neur_chan \
    --hmm hmm/per_domain/Neur_chan_LBD_REVISION.hmm

Outputs:

  • obim_hmm/hits.tbl — per-sequence tabular output
  • obim_hmm/hits.domtbl — per-domain tabular output
  • obim_hmm/hits_with_species.tsv — per-sequence hits joined to species metadata
Pick HMMs interactively. The HMM browser lets you filter the 190 HMMs by category (GPCR, VGIC, LGIC, …), tick a subset, and download as either a curl snippet or a zipped bundle.
Example 05 · custom search

Pass any flag to the underlying tool.

Every wrapper exposes the most-used flags by name and an --extra "ARGS" passthrough so anything we haven't surfaced is still reachable. The string is split on whitespace and appended verbatim: 

wrappers/mg_blast.sh -q q.fa -o out -d aa \
    --extra "-gapopen 11 -gapextend 1 -seg yes"

wrappers/mg_diamond.sh -q q.fa -o out \
    --extra "--masking 0 --comp-based-stats 1 --frameshift 15"

wrappers/mg_place.sh -q seeds.fa -o phylo \
    --iqtree-extra "-alrt 1000 -lbp 1000" \
    --search-extra "-soft_masking true"
Reproducibility. Whatever you pass via --extra also lands verbatim in the run.log, alongside the tool version that was used. Reviewers asking "what settings did you use?" get a single deterministic answer.
